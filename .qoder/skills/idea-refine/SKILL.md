# Idea Refine

Refines raw ideas into sharp, actionable concepts worth building through structured divergent and convergent thinking.

## How It Works

1. **Understand & Expand (Divergent):** Restate the idea, ask sharpening questions, and generate variations.
2. **Evaluate & Converge:** Cluster ideas, stress-test them, and surface hidden assumptions.
3. **Sharpen & Ship:** Produce a concrete markdown one-pager moving work forward.

## Process

### Phase 1: Understand & Expand
1. Restate as a "How Might We" problem statement
2. Ask 3-5 sharpening questions (who, success criteria, constraints, prior attempts)
3. Generate 5-8 idea variations using lenses: Inversion, Constraint Removal, Audience Shift, Combination, Simplification, 10x Version

### Phase 2: Evaluate & Converge
1. Cluster ideas into 2-3 distinct directions
2. Stress-test against: User Value, Feasibility, Differentiation
3. Surface hidden assumptions explicitly

### Phase 3: Sharpen & Ship
Produce a one-pager with:
- Problem Statement
- Recommended Direction
- Key Assumptions to Validate
- MVP Scope
- Not Doing list (the most valuable part)

## Anti-patterns
- Don't generate 20+ ideas — 5-8 well-considered variations are enough
- Don't be a yes-machine — push back on weak ideas
- Don't skip "who is this for"
- Don't produce a plan without surfacing assumptions

## Verification

- [ ] A clear "How Might We" problem statement exists
- [ ] The target user and success criteria are defined
- [ ] Multiple directions were explored
- [ ] Hidden assumptions are explicitly listed
- [ ] A "Not Doing" list makes trade-offs explicit
- [ ] The user confirmed the final direction before any implementation
